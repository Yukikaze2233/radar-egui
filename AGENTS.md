## 数据流

```
SDR (Python)
    │
    │ TCP 原始字节流
    │ 127.0.0.1:2000
    ▼
┌─────────────────┐
│   TCP 客户端     │  tokio 异步，自动重连
│   tcp_client.rs  │  buffer ≥ 200 字节 → 解析
└────────┬────────┘
         │
         │ Vec<u8>
         ▼
┌─────────────────┐
│   协议解析器      │  滑动窗口 cmd_id 扫描
│                  │  混合字节序 (BE + LE)
│   protocol.rs    │
└────────┬────────┘
         │
         │ Option<RoboMasterSignalInfo>
         ▼
┌─────────────────┐
│   共享状态        │  Arc<Mutex<RoboMasterSignalInfo>>
│   (app.rs)       │
└────────┬────────┘
         │
         ├──────────────────┬──────────────────┐
         ▼                  ▼                  ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│   小地图组件   │  │   状态面板    │  │   连接状态    │
│   (Painter)   │  │   (egui)     │  │   (顶栏)     │
└──────────────┘  └──────────────┘  └──────────────┘
```

## 模块职责

### main.rs
- 程序入口
- 初始化 egui 窗口 (1280×720)
- 初始化 env_logger

### protocol.rs
- `RoboMasterSignalInfo` 结构体 — 所有比赛状态字段
- `parse_signal(data: &[u8]) -> Option<RoboMasterSignalInfo>`
- 滑动窗口 cmd_id 扫描，与 Python 行为一致
- 边界条件单元测试

### tcp_client.rs
- `run_signal_client(addr, shared, shutdown)`
- tokio 异步 TCP 连接
- 缓冲区累积（阈值 200 字节）
- 连接断开自动重连
- 通过 watch channel 优雅关闭

### app.rs
- `RadarApp` 结构体 — egui 应用状态
- 后台线程启动 tokio 运行时
- 4 面板布局：顶栏、左侧小地图、中央状态面板
- 连接状态检测（5 秒超时）
- 数据更新重绘（100ms 间隔）

### widgets/minimap.rs
- `MinimapWidget` — 2D 战场可视化
- egui Painter 自定义绘制
- 6 个机器人以彩色圆点 + 标签显示
- 网格背景 + 十字准星
- Y 轴翻转（屏幕 Y 向下，世界 Y 向上）
- 坐标范围 ±3000（典型 RoboMaster 场地）

### widgets/panels.rs
- `StatusPanels` — 血量/弹药/经济/增益显示
- 血量：水平进度条，颜色编码（绿/黄/红）
- 弹药：数值网格
- 经济：剩余/总计 + 进度条
- 增益：每机器人增益明细表（5 个子字段）

## 关键设计决策

### 为什么用 egui 而不是 rerun 做实时渲染
- Rerun 是日志优先设计，不适合实时 HUD
- egui 即时模式：无状态同步开销
- 帧预算：100ms 重绘间隔（10 fps）对 HUD 足够
- Rerun 作为可选功能用于录制/回放

### 为什么用 Arc<Mutex<T>> 而不是 channel
- egui 每帧读取状态（10 fps）
- Channel 需要清空，会丢失最新值
- Mutex 竞争极小（1 线程写，1 线程读）
- 替代方案：watch channel — 更简单但灵活性低

### 为什么用滑动窗口解析器
- Python 解析器在每个字节偏移扫描 cmd_id
- 无固定帧分隔符 — 必须匹配此行为
- 未找到有效 cmd_id 时返回 None（优雅降级）
